metrics_analyser:
  description: >
    You are responsible for computing system metrics using the current
    monitor output and historical data.

    INPUT:
    - Current monitor output:
      {monitor_output}

    AVAILABLE TOOLS:

    1. Get Last Monitor Output
    2. Get Last Analysis Output

    3. Is Cube Moving
       Input: cube_velocity from monitor_output.cube.velocity

    4. Is Cube Getting Closer To Target
       Input:
       - current_distance = monitor_output.metrics.dist_cube_target
       - previous_distance = value from "Get Last Monitor Output"

    5. Get Current Target
       Input:
       - target = monitor_output.target

    INSTRUCTIONS:

    STEP 1:
    Extract cube velocity and use "Is Cube Moving"

    STEP 2:
    Retrieve previous monitor output and extract previous distance

    STEP 3:
    Use "Is Cube Getting Closer To Target" with:
      current_distance and previous_distance

    STEP 4:
    Use "Get Current Target" with monitor_output.target

    STEP 5:
    Combine ALL tool outputs into final JSON

    RULES:
    - Never compute manually what tools can compute
    - Never assume missing history
    - Output must strictly follow schema

  expected_output: >
    Return ONLY a JSON:

    {
      "cube_status": boolean,
      "cube_target_trend": "closer | farther | stable",
      "current_target": string
    }

  agent: analyser_agent


analysis_diagnosis:
  description: >
    Generate the final diagnosis using ONLY the output generated by
    Metrics Analyzer.

    ```
    INPUT:

    {metrics_analyser}

    IMPORTANT:

    - Metrics Analyzer has already performed all calculations.
    - Metrics Analyzer has already used any available historical data.
    - Do NOT retrieve history.
    - Do NOT call tools.
    - Do NOT recalculate metrics.
    - Do NOT invent values.

    Copy the following fields EXACTLY from {metrics_analyser}:

    - cube_status
    - cube_target_trend
    - current_target

    Your responsibility is only to interpret the meaning of the
    metrics and produce a diagnosis.

    Determine:

    - symptom
    - urgency
    - explanation
    - anomalies

    Diagnosis Rules:

    NORMAL OPERATION:

    If the metrics indicate expected behavior:

    - symptom = "none"
    - urgency = "low"
    - anomalies = []

    The explanation must clearly state why the cube behavior is
    considered normal.

    ABNORMAL OPERATION:

    If the metrics indicate unexpected behavior:

    - symptom must describe the problem.
    - urgency must be one of:
        - "low"
        - "medium"
        - "high"
    - anomalies must contain the detected issues.
    - explanation must justify the diagnosis using only the metrics
      received from Metrics Analyzer.

    Output Rules:

    - Return ONLY valid JSON.
    - Do not use markdown.
    - Do not add extra text.
    - Include all required fields.
    ```

  expected_output: >
    {
      "cube_status": true,
      "cube_target_trend": "closer",
      "current_target": "target_name",
      "symptom": "none",
      "urgency": "low",
      "explanation": "The cube is behaving as expected.",
      "anomalies": []
    }

  context:
    - metrics_analyser

  agent: analyser_agent
