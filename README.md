# Agentic MAPE-K for Robotic Arms

## Overview

This project presents an implementation of a MAPE-K (Monitor, Analyze, Plan, Execute over a Knowledge Base) architecture applied to the adaptive control of robotic arms, exploring concepts from Self-Adaptive Systems and Agent-Based Artificial Intelligence.

The main contribution of this research is the transformation of part of the traditional MAPE-K architecture into an agentic architecture, in which the Analysis (Analyzer) and Planning (Planner) components are implemented as intelligent agents capable of leveraging Large Language Models (LLMs), memory, and specialized tools to support the decision-making process.

The objective is to investigate how intelligent agents can make the adaptation process more flexible, interpretable, and reusable in dynamic robotic environments.

## Architecture

- Monitor
    Responsible for collecting the current state of the robotic environment, including the robot's position, the cube's state, the target, and execution metrics.

- Knowledge Base
    Stores historical execution information.
  
- Analyser
    - Deterministic Analysis
          Responsible for the initial processing of the data collected by the Monitor. This stage compares the current execution state with historical data stored in the Knowledge Base to derive objective metrics about the environment. It identifies the cube's movement along the vertical and horizontal axes, determines whether the cube is moving or stationary, and detects potential obstacle-induced blockages by combining obstacle information, cube displacement, and robot velocity through deterministic rules. The resulting metrics provide a structured representation of the environment, serving as input for the Analyzer Agent.
      
    - Analyser Agent
          The Analyser Agent receives the metrics produced by the deterministic analysis stage along with information stored in the Knowledge Base. Using a Large Language Model (LLM), the agent interprets the current state of the system, identifies potential anomalies, and generates a detailed diagnosis of the system's current condition.

- Planner Agent
    Receives the diagnosis produced by the Analyser and determines which adaptation strategy should be executed, considering the current state of the system and the information stored in the Knowledge Base.
  
- Executor
    Executes the plan generated by the Planner in the robotic environment.
  
## Application

The project was developed for experiments with robotic arms, where each cycle of the MAPE-K architecture receives information from the environment, performs an intelligent analysis, and automatically determines the most appropriate adaptation for the task being executed.

The monitored information includes, for example:

- Robot position;
- Joint velocities;
- Manipulated object position;
- Target position;
- Distances between the robot, the object, and the target;

These data are used by the agents to generate more comprehensive diagnoses and support decision-making throughout the task execution.

## How to Run

### 1. Clone the repository

```bash
git clone repository url

cd repository name
```

### 2. Install the dependencies

It is recommended to create a virtual environment.

```bash
python -m venv venv
```

Activate the virtual environment:

**Windows**

```bash
venv\Scripts\activate
```

**Linux/macOS**

```bash
source venv/bin/activate
```

Install the project dependencies:

```bash
pip install -r requirements.txt
```

### 3. Run the project

Simply execute the main script:

```bash
python src/main.py
```
